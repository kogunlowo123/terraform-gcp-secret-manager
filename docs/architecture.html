terraform-gcp-secret-manager

Enterprise Secrets Management on Google Cloud

Google Cloud Platform Terraform Module

Executive Summary

This Terraform module manages secrets throughout their lifecycle in Google Cloud Secret Manager. It supports automatic and user-managed replication strategies with per-region CMEK encryption, secret versioning with enable/disable controls, base64-encoded data, rotation policies, TTL and timestamp-based expiration, Pub/Sub event notifications, version aliasing, and IAM bindings with conditional access. The module enforces role validation for IAM bindings and project ID format validation.

Component Breakdown

Secrets (google_secret_manager_secret)

Core secret resources with labels, annotations, replication config (auto or user-managed with per-region CMEK), rotation policy (period + next time), TTL/expiration for auto-deletion, Pub/Sub topic notifications, and version aliases for named references to specific versions.

Secret Versions (google_secret_manager_secret_version)

Immutable secret data storage with enable/disable toggle. Supports base64-encoded data via is_secret_data_base64 flag with automatic decoding. Depends on parent secret creation.

IAM Bindings (google_secret_manager_secret_iam_member)

Per-secret IAM member bindings with role validation (secretAccessor, secretVersionAdder, secretVersionManager, admin, viewer, owner, editor). Supports conditional access with CEL expressions for time-bound or attribute-based access.

Replication Strategies

Two modes: (1) Automatic replication with optional CMEK encryption key, or (2) User-managed replication with explicit region replicas, each optionally encrypted with per-region KMS keys for data sovereignty compliance.

Industry Adaptation

Financial Services

User-managed replication with per-region CMEK. Rotation policies for credential cycling. Conditional IAM for time-limited access.

Healthcare

CMEK encryption for PHI. TTL-based expiration for temporary credentials. Pub/Sub notifications for audit logging.

SaaS Platforms

Version aliases for blue/green secret rotation. Base64 encoding for binary secrets. Auto replication for global availability.

DevOps / CI/CD

Programmatic secret versioning. Workload Identity for keyless access. Pub/Sub integration for secret change events.

Production Readiness Checklist

  • Enable CMEK encryption for all secrets
  • Configure rotation policies for credentials and API keys
  • Set TTL/expiration for temporary secrets
  • Apply least-privilege IAM (secretAccessor, not admin)
  • Use conditional access for sensitive secrets
  • Enable Pub/Sub notifications for audit trail
  • Use version aliases for zero-downtime rotation
  • Choose user-managed replication for data residency
  • Mark secret_versions as sensitive in Terraform state
